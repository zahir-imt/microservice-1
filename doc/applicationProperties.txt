In Spring Boot, the recommended naming convention for configuration properties is kebab-case in files and camelCase in Java classes

The Configuration Class Template
You use the @ConfigurationProperties annotation to bind external properties to a strongly typed Java object.
java

import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.context.annotation.Configuration;

@Configuration
@ConfigurationProperties(prefix = "app.service-config") // Use kebab-case for the prefix
public class ServiceConfig {

    private String apiKey;       // Binds to api-key
    private int timeoutSeconds;  // Binds to timeout-seconds
    private boolean enabled;     // Binds to enabled

    // Standard
}

Property Naming Conventions
=============================
Spring Boot uses Relaxed Binding, which allows for multiple formats in your configuration files, but there are "canonical" best practices to follow:

    In .properties or .yaml files: Use kebab-case (e.g., app.service-config.api-key). This is the industry-standard "canonical form" for Spring configuration.
    In Java Classes: Use standard camelCase for field names (e.g., private String apiKey).
    Environment Variables: Use UPPER_SNAKE_CASE (e.g., APP_SERVICECONFIG_APIKEY). Spring automatically maps these to your Java fields.

Why use @ConfigurationProperties?
=================================

    Type Safety: It validates your data types (e.g., ensuring a timeout property is actually an int).
    Validation: You can use standard JSR-303 annotations like @NotNull or @Min(10) directly on your fields.
